## 一、测评维度概览

本测试共 **4 个维度**，每个维度通过多道情境题测量，最终组合成 **4 位人格代码**（如 `ENJA`），对应 16 种人格类型。

| 维度 | 名称 | 左极 | 右极 | 题目数 |
|------|------|------|------|--------|
| 维度一 | 充电模式 | E（社交电源） | I（独行电池） | 7 |
| 维度二 | 脑回路模式 | S（细节侦探） | N（脑洞建筑师） | 4 |
| 维度三 | 执行节奏 | J（计划仙人） | P（DDL战神） | 6 |
| 维度四 | AI 协作模式 | A（AI 原生派） | H（古法科研派） | 10 |

**总计 27 题**，每题 4 个选项，分别对应从强左极到强右极的连续谱系。

## 六、计分规则

### 6.1 维度得分累加

每道题的选项带有 `scores` 分值，如 `{"E": 3}`、`{"I": 1}`。系统遍历用户所有答案，将同一维度的分值累加：

```
E_total = 所有选 E 的分数之和
I_total = 所有选 I 的分数之和
S_total = 所有选 S 的分数之和
N_total = 所有选 N 的分数之和
J_total = 所有选 J 的分数之和
P_total = 所有选 P 的分数之和
A_total = 所有选 A 的分数之和
H_total = 所有选 H 的分数之和
```

### 6.2 维度判定

每个维度比较两边得分，高者胜。同分默认取**左极**：

```
if E_total >= I_total → 取 E，否则取 I
if S_total >= N_total → 取 S，否则取 N
if J_total >= P_total → 取 J，否则取 P
if A_total >= H_total → 取 A，否则取 H
```

### 6.3 人格代码组合

将 4 个维度的获胜极按顺序组合成 4 位人格代码：

```
人格代码 = E/I + S/N + J/P + A/H
```

例如：`E` + `N` + `J` + `A` = `ENJA`（预言家）

### 6.4 结果查询

系统根据人格代码查询 `Personality` 表，返回对应的人格名称、称号、描述、特征维度、像素画等信息。

## 八、设计说明

### 8.1 场景取材

全部题目场景取材于**香港科技大学（广州）校园生活**，包括：
- 红鸟音乐节、社团摆摊、科广夜市
- 校园明星猫「秋秋」
- 早九课、期末大作业、期末考试月
- 跨学科项目、读论文、写代码、跑实验
- 做 PPT、写论文引言、论文润色

### 8.2 选项设计原则

1. **四选项连续谱系**：每道题的 A→B→C→D 构成从强左极到强右极的连续光谱，而非简单二选一。
2. **分值梯度**：强极 +3，弱极 +1，无中间值，保证区分度。
3. **校园梗与幽默**：选项语言轻松有趣，融入学生日常用语和自嘲，降低测试焦虑感。
4. **AI 维度特色**：维度四紧扣港科广「AI 深度融合教学科研」的校情，探讨「怎么用 AI」而非「用不用 AI」。

### 8.3 扩展性

内容运营者可通过「创作中心 → 制作问卷」功能：
- 从基础题库勾选题目（不修改原题）
- 添加个性化自定义题目
- 设置新的人格类型与对应解读
- 提交管理员审核通过后生成新测试模板
